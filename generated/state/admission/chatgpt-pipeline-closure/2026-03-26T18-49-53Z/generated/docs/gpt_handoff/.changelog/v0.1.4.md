# v0.1.4

Promoted the ChatGPT packet model from a temporary sidecar narrative to an operational packet pipeline with a first-class `problem_set` ingress contract.

Added explicit pipeline manifests for execution, validation, and gating so local tooling owns admission and realization while ChatGPT remains inside the artifact-authoring boundary.

Switched the validation surface to `schemas/chatgpt-packet/**` and made the `packet.review.request.json` plus `packet.review.decision.json` pair the canonical review boundary, with `packet.approval.json` retained only as transitional compatibility.

Clarified that `packet.definition.json` is the collapsed plan artifact and that regeneration is driven only by meaningful input, authority, schema, or output-contract changes.
