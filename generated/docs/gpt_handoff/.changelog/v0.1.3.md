# v0.1.3

Added `workflow_state` as the canonical packet lifecycle field and kept `review_state` as the narrower review-only substate.

Aligned packet state modeling with the workflow by defining explicit mapping rules between `workflow_state` and `review_state`, including the `REQUESTED`, `REALIZED_VIA_SCM_PATTERN`, and `CLOSED` lifecycle states.

Split the review lifecycle into a pre-decision `packet.review.request.json` artifact and a conditional human-issued `packet.review.decision.json` artifact, while keeping `packet.approval.json` as transitional compatibility only if retained.

Expanded the approval basis to include `control/scm.pattern/authority.manifest.json` directly and required both `authority_manifest_ref` and `authority_manifest_fingerprint` as trust-bound review basis fields.

Clarified that any semantic change to the authority manifest marks prior approval `STALE` and forces reapproval, while the root trust evidence is tracked separately from packet-local review state.
